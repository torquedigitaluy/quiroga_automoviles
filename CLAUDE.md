# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

A single-page marketing + inventory site for **Quiroga Automóviles**, a Uruguayan car dealership. Exported from Figma Make, now a Vite + React 18 + TypeScript app styled with Tailwind v4 and shadcn/ui. All UI copy is in Spanish; prices are USD/UYU with a configurable exchange rate (`tipoCambio`).

## Commands

```bash
pnpm i        # or npm i — installs deps (pnpm-workspace.yaml present, single-package workspace)
pnpm dev      # vite dev server
pnpm build    # vite build → dist/
```

There is **no test runner, no linter, and no typecheck script** configured. `tsc` is not wired into the build (Vite transpiles without type-checking). Do not invent test/lint commands.

## Single entry point

`index.html` is the Vite entry: a minimal shell that loads `/src/main.tsx` → `src/app/App.tsx`. The React app under `src/` is the only maintained implementation.

The original Figma Make single-file static prototype (all CSS/JS inlined) has been **retired** to `legacy/index.static.html`. It is no longer wired to anything and Vite does not build it — kept only for reference. Do not edit it expecting changes to appear in the app.

## Routing (src app)

`src/app/App.tsx` defines all routes via `react-router` v7:
- `/` — `LandingPage` (composes the section components: Navbar, Hero, Services, StockVehiculos, CalculadoraFinanciacion, Financiacion, Seguros, CotizadorVehiculo, Taller, Sucursales, Footer + WhatsApp FAB)
- `/autos` — full vehicle listing (`VehiculosPage`)
- `/admin/login` and `/admin` — admin CRUD (`AdminLogin` / `AdminPanel`, both exported from `pages/AdminPanel.tsx`)

## Data layer — Supabase, called directly from the browser

`src/app/contexts/VehiclesContext.tsx` is the **single source of truth** for vehicle data. It wraps the whole app and exposes `vehicles` (published only), `allVehicles` (admin), and CRUD methods. There is no separate API layer — the context calls Supabase directly from the client using the anon key.

- Client: `src/lib/supabase.ts`, configured from `utils/supabase/info.tsx` (**autogenerated — do not edit**; holds `projectId` and `publicAnonKey`).
- Table: `vehicles`. The DB row shape is `VehicleRow` in `src/lib/supabase.ts`; `rowToVehicle()` maps it to the UI `Vehicle` type (computes `name` from marca+modelo+version, derives `features`, builds `whatsappText`, orders cover image first).
- Media: uploaded to the public Supabase Storage bucket `vehicle-media` via `uploadMedia()`.
- After any mutation (`addVehicle`/`updateVehicle`/`deleteVehicle`), the context re-runs `load()` to refetch — no optimistic updates.
- `tipoCambio` (USD→UYU rate) is persisted in `localStorage` under `quiroga_tc` (default 43).

### One-time DB/bucket provisioning

`supabase/functions/server/index.tsx` is a Hono/Deno **Edge Function** (route prefix `make-server-f85fd53a`) with `/health` and a `/setup` endpoint that creates the `vehicles` table (with RLS policies) and the `vehicle-media` bucket using the service-role key. This is provisioning-only; runtime reads/writes go straight through the anon client, not this function. `src/lib/setupDb.ts` is a client-side stub that probes whether the table exists.

### Admin auth is not real auth

`pages/AdminPanel.tsx` checks a **hardcoded** `ADMIN_USER`/`ADMIN_PASS` and sets a `sessionStorage` flag (`quiroga_admin`). There is no server-side authorization — RLS on `vehicles` grants anon full access. Treat the admin gate as cosmetic.

## Conventions

- **Styling is mixed.** Components use Tailwind utility classes *and* large inline `style={{...}}` objects with hardcoded hex colors (brand blue `#1634D4`, near-black `#0d0d14`) and `'Poppins'` font. Match the surrounding component's style rather than imposing one approach.
- **`@/` aliases `src/`** (vite.config.ts).
- **`figma:asset/<file>` imports** resolve to `src/assets/<file>` via a custom Vite plugin (`figmaAssetResolver` in vite.config.ts). Real image/photo assets currently live in `src/imports/` and are imported by relative path.
- `assetsInclude` covers `.svg` and `.csv` for raw imports — do **not** add `.css`, `.ts`, or `.tsx` to it.
- shadcn/ui primitives live under `src/app/components/ui/` (generated; avoid hand-editing unless intentionally customizing).
- The React and Tailwind Vite plugins are both required by Figma Make even though Tailwind config is minimal — do not remove them.
